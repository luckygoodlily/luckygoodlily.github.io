Mysql linux安裝和指令

檢查安裝

透過Linux指令檢查是否已經安裝過mysql，執行命令。如果沒有任何顯示資訊，則說明未安裝，如果存在資訊，則執行卸載。

卸載MySQL

sudo apt-get autoremove --purge mysql-*

查看有哪些mysql的id還在跑停掉

ps -aef | grep mysql
kill掉id

安裝MySQL

三條指令請分別依序的安裝，分別為伺服端、客戶端以及開發中一些MySQL資料庫的函式庫，記得要在 sudo 管理者權限下才能安裝。

測試資料庫是否安裝成功

可以透過 netstat 指令來查看， netstat 的功用是顯示通訊協定統計資料以及目前的TCP/IP 網路連線，所以可以使用它來查看已安裝好的 MySQL 是否有連線監聽。

安裝指令如下：

sudo apt-get install net-tools
安裝完成後就能透過 netstat 指令來查看 MySQL 是否安裝成功：

sudo netstat -tap | grep mysql

設置 MySQL 允許遠端訪問

編輯 mysqld.cnf 文件這個文件定義了所有 MySQL 的配置參數。進入後將 bind-address = 127.0.0.1 註解起來，完成後鍵盤輸入 :wq 儲存並退出 vi。

sudo vi /etc/mysql/mysql.conf.d/mysqld.cnf

設定Mysql root 密碼

方法一 // 失敗

sudo vim /etc/mysql/debian.cnf

方法二 // 失敗

ALTER USER ‘root’@‘localhost’ IDENTIFIED BY ‘new_password’;

方法三

sudo mysql_secure_installation

輸入您選擇的密碼強度數字，然後按Enter。然後，係統將詢問您新的root密碼。在以下提示中輸入密碼，然後輸入re-enter。

然後，係統將顯示您提供的密碼強度，並詢問您是否要繼續輸入密碼。

為是輸入Y，然後按Enter。

係統現在將一個接一個地向您提出一係列問題，您可以根據對這些問題的回答來設置係統的安全性。

繼續回答問題：

第一個問題將詢問您是否要刪除匿名測試用戶。

建議輸入y。

第二個問題將詢問您是否要禁止從遠程係統進行root登錄。通常，這是您的選擇，因為對於安全係統，僅應允許root從本地主機連接。

第三個問題將詢問您是否要從係統中刪除名為“test”的默認MySQL數據庫，並刪除對其的訪問


輸入y刪除此測試數據庫。

為了使您的所有above-configured更改生效，係統需要重新加載特權表。輸入y，您的所有安全更改將被提交

停止Mysql 服務

systemctl stop mysql.service

重起 Mysql 服務

systemctl start mysql.service

重新啟動mysql

/etc/init.d/mysql stop
/etc/init.d/mysql start
or
/etc/init.d/mysql restart

啟動MySQL Shell

以sudo運行以下命令來啟動MySQL Shell：

sudo mysql

檢查MySQL用戶的身份驗證方法

在MySQL提示符下，輸入以下命令，該命令可讓您檢查所有MySQL帳戶當前正在使用的身份驗證方法/插件：

mysql> SELECT user,authentication_string,plugin,host FROM mysql.user;

更改root用戶的身份驗證方法

讓root用戶在MySQL上使用密碼進行身份驗證。為此，請運行以下命令，以使根由mysql_native_password標識。密碼需要包含英文大小寫和符號ex: ‘!Aa1235’

ALTER USER ‘root’@‘localhost’ IDENTIFIED WITH mysql_native_password BY ‘your password’;

重新載入授權表

現在是時候告訴服務器使用新的特權設置了。在MySQL提示中運行以下命令以重新加載授權表並注冊您的更改：

mysql> FLUSH PRIVILEGES;

重新檢查MySQL用戶的身份驗證方法

當您通過以下命令重新檢查MySQL用戶帳戶的身份驗證方法時，您將看到您的root用戶現在正在使用mysql_native_password插件進行身份驗證：

mysql> SELECT user,authentication_string,plugin,host FROM mysql.user;

可以透過root用戶已設置為通過安全密碼連接到MySQL Shell，您可以使用exit命令如下退出shell：

mysql> exit

測試MySQL是否已啟動並正在運行

查詢 mysql.service 狀態

systemctl status mysql.service


mysql.service最有可能自動運行。以下命令的輸出應驗證服務的活動狀態：

以root用戶身份連接到MySQL Admin並運行任何管理命令

MySQL Admin是一個客戶端，可讓您在MySQL上執行管理操作。讓我們通過它運行一個管理命令作為示例，以檢查係統是否正常運行，並且我們的根配置為使用它。

$ sudo mysqladmin -p -u root version

該命令旨在以root用戶身份連接到MySQL，提示輸入root密碼，然後返回MySQL admin的版本號。

開始使用Mysql

以root的方式進入Mysql

mysql -u root -p
輸入密碼後進入

列出 MySQL Server 上的資料庫

SHOW DATABASES;

建立資料庫

CREATE SCHEMA Monitor;

在某資料庫下建立資料表

列出資料庫的資料表

SHOW TABLES [FROM db_name]
SHOW TABLES FROM Monitor;

顯示資料表的欄位

SHOW COLUMNS FROM duration(tbl_name);
=>ERROR 1046 (3D000): No database selected
要旨定db_name
SHOW COLUMNS FROM duration From Monitor;
SHOW COLUMNS FROM tbl_name [FROM db_name]

列出資料表的欄位，提供比較詳細的訊息

SHOW FULL COLUMNS FROM duration;

顯示資料表內容

SELECT * FROM Monitor.duration;

(From db_name.tbl_name)
或著使用

use Monitor;// 轉換資料表
SELECT * FROM duration;

顯示只要符合特定條件的資料

select * From baseline where server=‘test’;
select * From baseline where server=‘test’ and TO_DAYS(query_date)=TO_DAYS(NOW());

清空資料表

TRUNCATE Table Monitor.duration;

刪除table

DROP TABLE baseline

group by 和 order by

group by分組查詢

select avg(age),city from users group by city
統計每個城市的平均年齡

select count(age),age,city from users group by city,age
計算各個城市每個年齡階段的人數

order by 排序

ORDER BY "欄位名稱1" [ASC or DESC],"欄位名稱2" [ASC or DESC],"欄位名稱3" [ASC or DESC],...;

select * from table order by field desc|asc
desc:降序
asc:升序
ex：

select * from users order by user_id desc limit 20
根據使用者ID降序排序前20條使用者資訊。（先排序，在取限制結果）

實際應用的例子

select count(url), url from Monitor.duration where server=‘svl’ group by url order by count(url) desc ;

搜尋結果輸出excel檔案

select * From baseline where server=‘bb’ into outfile ‘bb.csv’ FIELDS TERMINATED BY ‘,’ ENCLOSED BY ‘"’ LINES TERMINATED BY ‘\n’;
或是
select * From baseline where server=‘bb’ into outfile ‘bb.xls’;
輸出txt檔的話
select * From baseline where server=‘bb’ into outfile ‘bb.txt’;
