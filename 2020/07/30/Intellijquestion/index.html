Ubuntu18.04.4下Intellij IDEA不能输入中文的问题

發表於 2020-07-30

無法用中文輸入法這個問題是因為用 snap 包裝造成的，所以才會需要改執行腳本來強制允許輸入法的模組執行

設定檔位置

使用者偏好的東西

~/.config/JetBrains

安裝執行檔位置

intellij idea 是 snap 包裝的

先設root密碼，使可以設定 idea.sh
linux: $ sudo passwd
mac: $ sudo passwd root

snap下載的安裝執行檔都會在這裡

$ vi /snap/intellij-idea-community/current/bin/idea.sh
//修改不了

儲存後重新啟動

$ ~/snap是mount point

umake install path

首先安裝 Ubuntu Make 如果使用Ubuntu 16.04, 18.04 或更高版本, 可以使用以下指令安裝:

sudo apt install ubuntu-make
Once you have Ubuntu Make installed, you can use the command below to install IntelliJ IDEA Community edition:

安裝idea

umake ide idea

/home/lily/.local/share/umake/ide/idea

解除安裝

umake -r ide idea

重新裝snap的idea edge最新的idea

sudo snap install intellij-idea-community --classic --edge

apt裝法

找apt/deb的安裝檔

在Ubuntu和所有其他基於Debian的發行版中，apt軟件存儲庫在/etc/apt/sources.list文件中或目錄下的單獨文件中定義/etc/apt/sources.list.d/。

/etc/apt/sources.list.d/目錄內存儲庫文件的名稱必須以結尾.list。
/etc/apt/sources.list文件的常規語法採用以下格式：

deb http://repo.tld/ubuntu distro component...

  • 該行的第一個條目定義了存檔的類型。存檔類型可以是deb或deb-src。Deb表示存儲庫中包含.deb軟件包，而deb-src表示源軟件包。
  • 第二項是存儲庫URL。
  • 第三個條目指定分發代碼名稱，例如beaver，xenial等。
  • 最後的條目是存儲庫組件或類別。默認的Ubuntu存儲庫分為四個部分-主組件，受限組件，Universe和multiverse。通常，第三方存儲庫只有一個類別。

需要sudo訪問權限或root 用戶的身份登錄。

安裝add-apt-repository（未找到add-apt-repository命令）

add-apt-repository是一個Python腳本，可讓您將APT存儲庫添加到目錄中的一個/etc/apt/sources.list或單獨的文件中/etc/apt/sources.list.d。該命令還可以用於除去已經存在的存儲庫。

如果add-apt-repository您的系統上沒有，您將收到一條錯誤消息，提示“找不到add-apt-repository命令”。

該add-apt-repository實用程序包含在software-properties-common包裝中。要安裝它，請運行以下命令：

使用以下方式添加存儲庫 add-apt-repository

該add-apt-repository命令的基本語法如下：

add-apt-repository [options] repository

可以通過此PPA獲得社區版本。PPA將下載IDE，因此再次安裝時間將取決於您的Internet連接
